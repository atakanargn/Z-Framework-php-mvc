2.1. Kullanıcı
Açıklama; Kullanıcı işlemleri için Giriş/Çıkış işlemleri

# Kullanıcı Direkt Giriş

* Direkt User Modelinden dönen TEK veriyi 1. parametre olarak girildiğinde kullanıcıya giriş yapacaktır. 
                Auth::login($user);

# Kullanıcı API olarak giriş yapacaksa

* Bu seçeneceği kullanacağınızı sanmam ama yine de anlatayım.
* URL'de İstekler dizesinde ?user_token=token şeklinde girdiğinizde tokeni eşleşen kullanıcı tek seferlik şekilde giriş yapmış olacaktır. 
                Auth::api_login($token);

# Kullanıcı Çıkış

* Geçerli kullanıcının oturumunu sonlandırmak için kullanılır. 
                Auth::logout();

# Kullanıcı Giriş yapmış mı?

* Eğer kullanıcı giriş yapmışsa TRUE eğer yapmamış ise FALSE değerini döndürecektir. 
                Auth::check()

# Geçerli kullanıcının bilgilerini almak

* Bize geçerli kullanıcının verilerini döndürecektir.
Performans Notu: Bu methodu bir kere çağırdığınız zaman istediğiniz kadar çağırabilirsiniz çünkü her seferinde tekrar çekmek yerine 1 kere çekip cache de bekletiyor. 
                Auth::user()

                # Kullanım olarak:
                echo Auth::user()['id'];

# Kullanıcı İstek ile giriş yaptırmak

* Giriş yaparken istenen bilgileri giriyoruz ve doğru olan ile eşleşen kullanıcı giriş yapabiliyor. 
                Auth::attempt(array)

                # Örnek Kullanım

                if (Auth::attempt(['username' => request('username'), 'password' => request('password')])) {
                    echo "Kullanıcı giriş yaptı!";
                } else {
                    echo "Kullanıcı adı veya şifre yanlış!";
                }

# Geçerli kullanıcının id değerini almak

* Giriş yapmış kullanıcının id değerini bize verir. 
                Auth::id()
